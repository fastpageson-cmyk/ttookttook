# 빌드 브리프 — 똑똑 웹사이트 프로토타입

## 1. 한 줄 목표

사회초년생이 아무 지식 없이 투자를 해보고 실패를 겪은 뒤(0단계), 거시경제 강의와 실습을 통해 왜 실패했는지 배우는(1주차) 흐름을, 목업 데이터로 동작하는 웹사이트 프로토타입으로 만든다.

## 2. 스코프 (반드시 지킬 것)

- **플랫폼**: 웹사이트. 반응형(모바일 화면에서도 쓸 수 있게)이면 좋지만 네이티브 앱은 만들지 않는다.
- **범위**: 0단계(사전진단 + 블라인드 자유투자 + AI 매매 실수 리포트) + 1주차(거시경제 강의 + 실습 3종 + 퀴즈)까지만 만든다. 2주차 이후는 이번 빌드에 포함하지 않는다.
- **회원가입/로그인 없음**. 시작 화면에서 닉네임 정도만 입력하고 바로 진행한다.
- **백엔드/DB 없음**. 모든 상태는 프론트엔드에서 관리한다(구체적인 저장 방식은 자유 — 새로고침 시 유지할지 여부도 알아서 판단).
- **데이터는 전부 목업**이다. `콘텐츠/mock-data/` 폴더의 JSON을 그대로 쓰면 된다. 실제 시세 데이터를 찾으러 다니지 않아도 된다.
- **기술 스택은 자유롭게 판단**해라(React든 순수 HTML/JS든, 팀이 정하지 않고 위임했다).
- **디자인 시스템은 없다.** 색상·폰트·톤앤매너는 자유롭게 정해도 된다. 다만 학교 토이 프로젝트이자 금융 교육 서비스이므로, 지나치게 화려하기보다 신뢰감 있고 차분한 톤을 권장한다.

## 3. 문서 읽는 순서

1. [[기본적인 컨셉과 개요.]] — 서비스가 왜 이런 구조인지(맥락)

3. [[화면기획서]] — 화면 12개(3-1~3-12)의 구성요소·인터랙션·예외 상황. **이게 실제 구현 스펙에 가장 가깝다.**
4. [[데이터모델]] — 프론트엔드 state 구조(TypeScript 인터페이스로 정의)
5. `콘텐츠/1주차_거시경제와_투자의_이유.md` — 1주차 강의 카드 10개, 실습 3종 스크립트, 퀴즈 5문항 (실제 텍스트 원고, 그대로 화면에 넣으면 됨)
6. `콘텐츠/사전진단_20문항.md` — 사전 진단 문항 20개 (그대로 사용)

## 4. 목업 데이터 사용법

`콘텐츠/mock-data/` 폴더에 3개 JSON이 있다.

- `rates.json`: 기준금리 시계열 520주(약 10년). 121~220주 구간이 "긴축기(금리 인상)"로, 0단계 시뮬레이션과 1주차 실습1(0단계 복기)에서 이 구간을 반드시 보여줘야 한다.
- `stocks.json`: 매매 가능한 종목 8개 + 지수ETF 1개, 520주 주간 종가. 0단계 자유투자 화면([[화면기획서]] 3-5)에서 이 데이터를 그대로 쓰면 된다.
- `sp500_scenario.json`: 1주차 실습2(금리 인하기 가이드 리플레이, [[화면기획서]] 3-10)용 별도 52주 시나리오. 실제 S&P500이 아니라 연출된 데이터라는 점을 화면에도 작게 표기하면 좋다(신뢰성 문제 예방).

세 파일 모두 최상단에 `disclaimer` 필드로 "실제 데이터 아님"이 명시돼 있다. 화면 어딘가(예: 푸터나 리포트 하단)에 "본 시뮬레이션은 학습용 가상 데이터를 사용합니다" 같은 한 줄 고지를 넣는 걸 권장한다.

## 5. AI 기능 처리 방침 — 반드시 읽을 것

[[화면기획서]] 3-7 "AI 매매 실수 리포트"는 서비스에서 유일하게 AI를 쓰기로 한 화면이다. 하지만 이번 **프로토타입 단계에서는 실제 LLM API를 호출하지 말고, 규칙 기반(rule-based) 로직으로 구현**해라. 이유:

- 로그인/백엔드가 없는 순수 프론트엔드 프로토타입이라, 실제 LLM을 호출하려면 API 키를 클라이언트에 노출하거나 별도 백엔드가 필요한데 둘 다 이번 스코프 밖이다.
- 매매 로그(`Trade[]`)에서 "가장 손실이 컸던 매매 3건"을 정렬해서 뽑아내고, 아래 같은 **템플릿 문장**에 실제 숫자를 채워 넣는 방식으로 충분히 같은 화면을 만들 수 있다.

```
"{week}주차에 {stockCode}를 {price}원에 매수했습니다. 이후 가격이 {changePercent}% 하락했을 때도 매도하지 않아, 최종적으로 {lossAmount}원의 손실이 발생했습니다."
```

- 데이터모델의 `AIMistakeReport`/`MistakeReportItem` 타입 구조를 그대로 쓰면 나중에 실제 LLM 호출로 교체할 때도 인터페이스가 바뀌지 않는다. 즉, **함수 시그니처는 AI를 쓰는 것처럼 만들되 내부 구현만 규칙 기반으로 채워라.** (예: `generateMistakeReport(trades: Trade[]): AIMistakeReport` 형태로 분리해두면, 나중에 이 함수 내부만 실제 API 호출로 바꾸면 된다.)

## 6. 완료 기준 (Acceptance Criteria)

아래가 전부 동작하면 이번 빌드는 완료다.

1. 닉네임 입력 → 사전 진단 20문항 응시 → 점수와 또래 평균(62.6점) 비교 화면이 보인다
2. 0단계 화면에서 목업 종목 8개 + 지수ETF를 매수/매도할 수 있고, 시간을 진행시키며 520주 중 원하는 시점까지 플레이할 수 있다(전부 다 플레이하지 않고 중간에 끝내는 것도 허용)
3. 0단계를 마치면 종료 리포트(수익률, MDD, 매매횟수, 수수료 총액, 지수ETF 대비 비교)가 나온다
4. AI 매매 실수 리포트(규칙 기반)가 실제 매매 로그를 기반으로 3건 생성되어 보인다
5. 1주차 강의 카드 10개를 순서대로 볼 수 있다
6. 1주차 실습 3종(0단계 복기, 금리 인하기 리플레이, 한미 금리차 위젯)이 각각 인터랙션 가능하다
7. 확인 퀴즈 5문항을 풀 수 있고, 통과 기준(잠정 4/5 이상 — 바꿔도 됨)을 만족해야 "완료" 표시가 뜬다
8. 전체 흐름(0단계 → AI 리포트 → 1주차 강의 → 실습 → 퀴즈)이 하나의 사이트 안에서 끊기지 않고 이어진다

## 7. 이번 빌드에서 하지 않아도 되는 것 (Out of scope)

- 2주차~6주차 콘텐츠/화면 (아직 콘텐츠 자체가 없음)
- 실제 회원가입/로그인/비밀번호
- 실제 DB, 서버, 배포 인프라 고민
- 실제 LLM API 연동 (5번 항목 참고 — 규칙 기반으로 대체)
- 실제 증권사 데이터 연동
- 결제, 유료화 관련 기능
- 법적 고지/약관 문구의 법률적 완성도 (학교 토이 프로젝트이므로 형식적인 안내 문구 정도면 충분)

## 8. 아직 정해지지 않은 것 (Claude Code가 알아서 정해도 되는 것)

- 퀴즈 통과 기준 점수 (잠정 4/5)
- 새로고침 시 진행 상태 유지 여부 및 저장 방식
- 구체적인 색상·폰트·컴포넌트 스타일
- 차트 라이브러리 선택

이 항목들은 팀 확인 없이 Claude Code가 합리적으로 판단해서 진행해도 된다.
