# 목업 데이터 안내

이 폴더의 JSON은 프론트엔드 프로토타입이 "일단 동작하게" 만들기 위한 합성(synthetic) 데이터입니다. 실제 과거 시세가 아니라, 금리 레짐에 따라 그럴듯하게 움직이도록 설계한 가짜 데이터입니다. 실제 데이터 소스가 확정되면 이 파일들을 같은 구조로 교체하면 됩니다. `generate_mock_data.py`를 실행하면 동일한 구조로 재생성됩니다(시드 고정, 재실행해도 같은 값).

## 파일 목록

### rates.json
0단계·1주차에서 쓰는 기준금리 시계열. 총 520주(약 10년), 주 단위.

```
{
  "meta": { "disclaimer": "...", "totalWeeks": 520 },
  "weeklyRates": [
    { "week": 1, "baseRate": 2.0, "phase": "완화 유지" },
    ...
    { "week": 145, "baseRate": 3.6, "phase": "긴축기 (금리 인상)" },
    ...
  ]
}
```

레짐 구간: 1~120주 완화 유지(2.0%) → 121~220주 긴축기(2.0%→5.0% 인상) → 221~300주 고금리 유지(5.0%) → 301~420주 완화 전환(5.0%→2.5% 인하) → 421~520주 저금리 유지(2.5%). **121~220주 긴축기 구간이 [[화면기획서]] 3-5에서 요구한 "뚜렷한 금리 변동기"입니다.** 0단계 시뮬레이션 기간 중 이 구간이 반드시 포함되도록 프론트엔드에서 활용하세요.

### stocks.json
0단계 자유투자에서 매매 가능한 종목 8개 + 지수 ETF 1개. 종목별 주간 종가, 총 520주.

```
{
  "meta": { "disclaimer": "...", "totalWeeks": 520, "unit": "주간 종가, 임의 통화 단위" },
  "stocks": [
    { "code": "A전자", "type": "성장주", "prices": [62000, 62150, ...] },
    ...
  ],
  "indexEtf": { "code": "지수ETF", "type": "지수 추종", "prices": [10000, ...] }
}
```

`prices` 배열의 인덱스 0이 1주차, 인덱스 519가 520주차입니다. `rates.json`의 `week` 필드와 인덱스+1로 매칭됩니다.

종목별 성격(화면기획서 3-5 "익명화된 종목" 요구사항에 맞춰 A전자~H반도체로 명명):

| 코드 | 성격 | 10년 배수(이번 생성 결과) |
|---|---|---|
| A전자 | 성장주 | 1.24배 |
| B바이오 | 고변동 테마주 | 0.32배 (큰 손실) |
| C케미칼 | 경기민감주 | 1.24배 |
| D플랫폼 | 성장주 | 0.22배 (큰 손실) |
| E에너지 | 원자재 연동 | 3.95배 |
| F리테일 | 경기방어주 | 2.8배 |
| G헬스케어 | 방어주 | 2.83배 |
| H반도체 | 고베타 성장주 | 0.47배 |
| 지수ETF | 분산 투자 | 1.94배 |

개별 종목 간 편차를 의도적으로 크게 뒀습니다(승자 3.95배 vs 패자 0.22배). 0단계에서 무지식 상태로 아무 종목이나 골라 투자하면 지수ETF보다 나쁜 결과를 낼 확률이 높게 설계된 것 — "무지식 투자는 위험하다"는 0단계의 설계 의도와 "개별 종목 선정보다 분산이 유리하다"는 5주차 ETF 레슨 모두를 이 데이터 하나로 뒷받침합니다.

### sp500_scenario.json — ✅ 실제 데이터 (2026-07-22 교체)
1주차 실습2(금리 인하기 가이드 리플레이)용. **이 파일만 유일하게 실제 시장 데이터입니다.**
이전에는 연출된 가상 시나리오였으나, 2026-07-22에 **2020년 실제 데이터로 교체**했습니다.

- **출처**: FRED (Federal Reserve Bank of St. Louis) — `SP500`(S&P 500 종가), `DFF`(미국 실효 연방기금금리)
- **생성**: `python3 fetch_sp500_2020.py` (API 키 불필요, 재실행하면 같은 결과)
- **리샘플링**: 일별 → ISO 주차별 마지막 거래일 종가, 52주

실제 2020년의 기록이 그대로 담겨 있습니다.

| 항목 | 값 |
|---|---|
| 시작(1주차) | 3,234.85 |
| 고점 대비 최대 낙폭 | **-31.8%** (저점 2,304.92, 12주차) |
| 연간 수익률 | **+14.5%** (연말 3,703.06) |
| 기준금리 | 1.55% → **0.09%** (3/3 -0.5%p, 3/15 -1.0%p 긴급 인하) |

```
{
  "meta": { "source": "FRED ...", "weeks": 52, "year": 2020,
            "startIndexValue": 3234.85, "maxDrawdownPct": -31.8, ... },
  "weeklyIndex": [3234.85, 3265.35, ...],
  "weeklyBaseRate": [1.55, 1.58, ...]
}
```

⚠️ 실습2의 체크포인트(12주차 "금리 방향", 34주차 "지수 반등 여부")는 실제 데이터로도
정답이 그대로 성립함을 확인했습니다. 데이터를 다른 연도로 바꾸면 체크포인트 주차와 정답을 반드시 재검증할 것.

### fundamentals.json — ✅ 실제 데이터 (5주차 기업 분석, DART)

5주차(숫자를 보고 고르기)용 실명 기업의 실제 재무지표·공시. 자세한 계산법은 스크립트 상단 주석 참고.

- **출처**: DART 오픈API(사업보고서 재무제표) + FinanceDataReader(주가·상장주식수)
- **지표**: PER=시가총액/당기순이익, PBR=시가총액/자본총계, 영업이익률=영업이익/매출액, 업종평균PER=비교군 중앙값
- **🔑 키**: `DART_API_KEY`는 프로젝트 루트 `.env.local`(gitignore·chmod600)에서만 읽는다. 코드·출력 JSON·번들에 절대 넣지 않는다(정적 SPA라 번들은 공개됨).

**생성 / 연 1회 갱신** — 사업보고서는 다음 해 3월 말 공시되므로, 4월 이후에 직전 연도로 재실행한다:

```bash
python3 fetch_dart_fundamentals.py              # 기본 = 2024 사업보고서
python3 fetch_dart_fundamentals.py --year 2025  # 갱신: 새 사업연도
cp fundamentals.json ../../../src/data/         # 산출물을 앱으로 복사
```

주가 조회 종료일은 자동으로 '오늘'이라(`WINDOW_END`) 코드를 매년 손댈 필요가 없다. 재현이 필요하면
`WINDOW_END=YYYY-MM-DD`로 고정한다. **연도를 바꾸면 5주차 교안의 주장·반례(예: SK하이닉스 저PER,
C케미칼 가치함정)가 새 숫자로도 성립하는지 반드시 재검증할 것.**
