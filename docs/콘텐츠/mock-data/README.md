# 목업 데이터 안내

이 폴더의 JSON은 프론트엔드 프로토타입이 "일단 동작하게" 만들기 위한 합성(synthetic) 데이터입니다. 실제 과거 시세가 아니라, 금리 레짐에 따라 그럴듯하게 움직이도록 설계한 가짜 데이터입니다. 실제 데이터 소스가 확정되면 이 파일들을 같은 구조로 교체하면 됩니다. `generate_mock_data.py`를 실행하면 동일한 구조로 재생성됩니다(시드 고정, 재실행해도 같은 값).

## 파일 목록

### rates.json
0단계·1주차에서 쓰는 기준금리 시계열. 총 520주(약 10년), 주 단위.

```
{
  "meta": { "disclaimer": "...", "totalWeeks": 520 },
  "weeklyRates": [
    { "week": 1, "baseRate": 2.0, "phase": "완화 유지" },
    ...
    { "week": 145, "baseRate": 3.6, "phase": "긴축기 (금리 인상)" },
    ...
  ]
}
```

레짐 구간: 1~120주 완화 유지(2.0%) → 121~220주 긴축기(2.0%→5.0% 인상) → 221~300주 고금리 유지(5.0%) → 301~420주 완화 전환(5.0%→2.5% 인하) → 421~520주 저금리 유지(2.5%). **121~220주 긴축기 구간이 [[화면기획서]] 3-5에서 요구한 "뚜렷한 금리 변동기"입니다.** 0단계 시뮬레이션 기간 중 이 구간이 반드시 포함되도록 프론트엔드에서 활용하세요.

### stocks.json — ✅ 실제 데이터 (2026-07-22 종목 재선정)
0단계 자유투자에서 매매 가능한 종목 8개 + KOSPI 지수. 실제 주간 종가(원), 총 520주(2016-07~2026-07).
출처 FinanceDataReader(한국거래소 공개 시세, 수정주가). `prices[0]`=1주차, `prices[519]`=520주차.

**종목 선정 의도(중요):** 0단계는 "무지식 상태의 단일 종목 몰빵/단타는 예측 불가능한 고변동이고
대부분 지수를 못 이긴다"를 체감시키는 화면이다. 이전 구성(삼성전자·SK하이닉스 등)은 **SK하이닉스가
10년 +5,255%**라 "몰빵=개꿀"이라는 반대 교훈을 줬다. 그래서 **화제였던 대형·테마주 8종목**으로 교체했다.

| 종목 | 성격(type) | 10년 수익률 | MDD | 몰빵 최악 |
|---|---|---|---|---|
| 씨젠 | 진단키트 테마주 | +56% | -88% | 자산 12% |
| 신풍제약 | 코로나 치료제 테마주 | +11% | **-96%** | 자산 4% |
| HMM | 해운 경기순환주 | +112% | -74% | 자산 26% |
| 카카오 | 플랫폼 성장주 | +97% | -79% | 자산 21% |
| 아모레퍼시픽 | 중국 소비 테마주 | -68% | -78% | 자산 23% |
| 엔씨소프트 | 게임 성장주 | -11% | -87% | 자산 14% |
| 두산에너빌리티 | 원전·중공업주 | +248% | -90% | 자산 10% |
| 삼성SDI | 2차전지 성장주 | +321% | -80% | 자산 20% |
| **KOSPI** | 지수 | **+238%** | -39% | — |

**정직성 원칙:** 승자만 빼고 패자만 넣지 않았다. 8종목 중 **6종목이 KOSPI(+238%)에 미달**하지만,
2종목(두산·삼성SDI)은 지수를 이겼다 — 단, **-80~90% 폭락을 견뎌야** 했다. 즉 '몰빵=무조건 손실'(거짓)이
아니라 '몰빵=고변동·예측 불가'(참)를 가르친다. 전 종목이 -74%~-96% 낙폭을 겪어, 고점 추격매수 시
파산(총자산<30%)이 발동한다. 실측 예: 신풍제약 초기 몰빵 → 2.8억까지 급등 후 전액 반납, 최종 +11%지만
MDD -96%, "KOSPI 적립이었다면 +169%"(157.8%p 차이). 특정 종목 추천이 아니며 과거 수익률은 미래를 보장하지 않는다.

**재선정/재생성:** `fetch_market_data.py` 상단 `STOCKS` 리스트 수정 후 재실행(rates.json은 KOSPI 날짜
기준이라 종목만 바꾸면 동일하게 재생성됨). 종목 변경 시 각 종목의 몰빵 최악 손실이 파산 임계값(-70%)에
걸리는지, 6/8이 지수에 미달하는지 확인할 것.

### fundamentals.json — ✅ 실제 데이터 (5주차 기업 분석, DART)

5주차(숫자를 보고 고르기)용 실명 기업의 실제 재무지표·공시. 자세한 계산법은 스크립트 상단 주석 참고.

- **출처**: DART 오픈API(사업보고서 재무제표) + FinanceDataReader(주가·상장주식수)
- **지표**: PER=시가총액/당기순이익, PBR=시가총액/자본총계, 영업이익률=영업이익/매출액, 업종평균PER=비교군 중앙값
- **🔑 키**: `DART_API_KEY`는 프로젝트 루트 `.env.local`(gitignore·chmod600)에서만 읽는다. 코드·출력 JSON·번들에 절대 넣지 않는다(정적 SPA라 번들은 공개됨).

**생성 / 연 1회 갱신** — 사업보고서는 다음 해 3월 말 공시되므로, 4월 이후에 직전 연도로 재실행한다:

```bash
python3 fetch_dart_fundamentals.py              # 기본 = 2024 사업보고서
python3 fetch_dart_fundamentals.py --year 2025  # 갱신: 새 사업연도
cp fundamentals.json ../../../src/data/         # 산출물을 앱으로 복사
```

주가 조회 종료일은 자동으로 '오늘'이라(`WINDOW_END`) 코드를 매년 손댈 필요가 없다. 재현이 필요하면
`WINDOW_END=YYYY-MM-DD`로 고정한다. **연도를 바꾸면 5주차 교안의 주장·반례(예: SK하이닉스 저PER,
C케미칼 가치함정)가 새 숫자로도 성립하는지 반드시 재검증할 것.**
